The practical workflow

1. First run

Your Super agent performs the task on a real computer: logging in, navigating UI, handling edge cases.

2. Action trace

The verified steps are stored as a reusable action trace rather than a brittle script.

3. Cache reuse

On the next run, Super replays the cached steps, skipping redundant reasoning.

4. Safer repetition

If the UI changes, Super can fall back to live computer use and update the cache.
